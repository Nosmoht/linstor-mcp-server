# Code Of Conduct

This project expects contributors to keep collaboration direct, respectful, and
evidence-based.

## Expected Behavior

- Discuss technical decisions with concrete reasoning and reproducible facts.
- Keep review comments focused on behavior, safety, maintainability, and user
  impact.
- Treat operators, contributors, and reviewers with respect, even during
  disagreement.
- Prefer clarification over assumption when a change could affect safety or
  production behavior.

## Unacceptable Behavior

- Personal attacks, insults, harassment, or intimidation.
- Dismissing safety concerns without technical justification.
- Sharing secrets, credentials, or sensitive cluster details in issues, pull
  requests, or logs.
- Intentionally disruptive behavior in project discussions or review threads.

## Reporting

Report conduct issues privately to the repository owner. Include links,
timestamps, and any relevant context so the issue can be reviewed quickly.
